# 批量导入导出功能使用说明

## 功能概述

系统现已支持批量导入和导出水电费账单数据到JSON文件。

## 使用方法

### 1. 批量导入JSON数据

**API端点：** `POST /api/v1/billing/import`

**请求示例：**
```bash
curl -X POST http://localhost:8080/api/v1/billing/import \
  -H "Content-Type: application/json" \
  -d @sample_import.json
```

**JSON格式示例：**
```json
[
  {
    "roomNumber": "A2",
    "currentWater": 2498,
    "previousWater": 2491,
    "waterAdjustment": 1,
    "waterPrice": 4.3,
    "currentElectric": 6084,
    "previousElectric": 6567,
    "electricAdjustment": 10,
    "electricPrice": 0.72,
    "managementFee": 22,
    "billingMonth": "2025年10月"
  }
]
```

**返回示例：**
```json
{
  "success": true,
  "message": "成功导入记录",
  "count": 10
}
```

### 2. 导出所有数据到JSON文件

**API端点：** `GET /api/v1/billing/export`

**请求示例：**
```bash
# 使用默认文件名
curl http://localhost:8080/api/v1/billing/export

# 指定文件名
curl "http://localhost:8080/api/v1/billing/export?filename=exports/my_export.json"
```

**返回示例：**
```json
{
  "success": true,
  "message": "导出成功",
  "file": "exports/billing_export.json"
}
```

## 字段说明

| 字段名 | 类型 | 说明 | 是否必填 |
|--------|------|------|----------|
| roomNumber | string | 房间号/住户编号 | 是 |
| currentWater | number | 本月水表读数 | 是 |
| previousWater | number | 上月水表读数 | 是 |
| waterAdjustment | number | 水表补差 | 否 (默认0) |
| waterPrice | number | 水单价 | 是 |
| currentElectric | number | 本月电表读数 | 是 |
| previousElectric | number | 上月电表读数 | 是 |
| electricAdjustment | number | 电表补差 | 否 (默认0) |
| electricPrice | number | 电单价 | 是 |
| managementFee | number | 管理费 | 是 |
| billingMonth | string | 账单月份 | 是 |

## 注意事项

1. **自动计算**：导入时系统会自动计算以下字段：
   - 用水量 (waterUsage)
   - 用电量 (electricUsage)
   - 水费 (totalWaterCost)
   - 电费 (totalElectricCost)
   - 总费用 (totalCost)

2. **ID自动分配**：每条记录的ID会由系统自动分配，无需在JSON中提供

3. **时间戳**：创建时间和更新时间会自动设置为当前时间

4. **数据持久化**：导入的数据会立即保存到 `data/billing_records.json` 文件中

## 示例文件

项目中已包含 `sample_import.json` 示例文件，您可以参考该文件的格式准备自己的导入数据。

## 错误处理

如果导入失败，系统会返回详细的错误信息：
```json
{
  "success": false,
  "error": "无效的JSON格式: ..."
}
```

常见错误：
- JSON格式不正确
- 必填字段缺失
- 数据类型不匹配
