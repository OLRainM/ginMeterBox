# 批量报表分辨率优化说明

## 📅 优化日期
2025年11月29日

---

## 🎯 优化目标

解决批量生成报表时，不同数量记录的显示适配问题，确保：
1. 少量记录时卡片足够大，信息清晰
2. 大量记录时合理布局，避免图片过大
3. 所有元素按比例自适应缩放
4. 支持多种分辨率和设备查看

---

## 🔧 优化内容

### 1. 智能布局策略

根据记录数量自动调整布局：

| 记录数量 | 卡片尺寸 | 每行数量 | 适用场景 |
|---------|---------|---------|---------|
| 1-2条 | 520×940 | 动态（1或2） | 单户或双户账单 |
| 3-6条 | 520×940 | 2个/行 | 小规模批量 |
| 7-12条 | 420×800 | 3个/行 | 中等批量 |
| 13+条 | 350×700 | 4个/行 | 大规模批量 |

**代码实现：**
```go
// 智能布局策略
if recordCount <= 2 {
    // 1-2条记录：大卡片，横向排列
    cardWidth = 520
    cardHeight = 940
    cardsPerRow = recordCount
} else if recordCount <= 6 {
    // 3-6条记录：中等卡片，每行2个
    cardWidth = 520
    cardHeight = 940
    cardsPerRow = 2
} else if recordCount <= 12 {
    // 7-12条记录：小卡片，每行3个
    cardWidth = 420
    cardHeight = 800
    cardsPerRow = 3
} else {
    // 13+条记录：更小卡片，每行4个
    cardWidth = 350
    cardHeight = 700
    cardsPerRow = 4
}
```

---

### 2. 高度限制保护

防止极大量记录导致图片过大：

```go
// 限制最大高度，避免图片过大（最大30000像素）
maxHeight := 30000
if totalHeight > maxHeight {
    // 如果超过最大高度，减少卡片高度
    scale := float64(maxHeight-headerHeight-(rows+1)*padding) / float64(rows*cardHeight)
    cardHeight = int(float64(cardHeight) * scale)
    totalHeight = headerHeight + rows*cardHeight + (rows+1)*padding
}
```

**保护机制：**
- 最大高度：30000像素
- 超过时自动缩放卡片高度
- 保持布局比例不变

---

### 3. 自适应字体大小

标题字体根据总宽度调整：

```go
// 字体大小根据宽度调整
titleSize := 42
subtitleSize := 18
if totalWidth < 800 {
    titleSize = 32
    subtitleSize = 14
} else if totalWidth > 2000 {
    titleSize = 52
    subtitleSize = 22
}
```

**效果：**
- 窄图片：小字体（32/14）
- 中等图片：标准字体（42/18）
- 宽图片：大字体（52/22）

---

### 4. 卡片元素缩放

所有卡片内元素按比例缩放：

```go
// 根据卡片宽度计算缩放因子（基准宽度520）
scale := float64(width) / 520.0

// 自适应字体大小
titleFontSize := int(24 * scale)
subtitleFontSize := int(12 * scale)
headerFontSize := int(20 * scale)
smallFontSize := int(11 * scale)
sectionTitleFontSize := int(14 * scale)
infoFontSize := int(12 * scale)

// 自适应间距和尺寸
padding := 20.0 * scale
headerBarHeight := 80.0 * scale
cornerRadius := 15.0 * scale
sectionCornerRadius := 8.0 * scale
lineHeight := 25.0 * scale
```

**缩放元素：**
- ✅ 字体大小
- ✅ 内边距
- ✅ 圆角半径
- ✅ 行高
- ✅ 卡片各部分高度
- ✅ 左右边距

dc.DrawStringAnchored(fmt.Sprintf("账单月份：%s  |  共 %d 条记录  |  布局：%d×%d  |  生成时间：%s",

## 🔍 技术细节

### 缩放计算公式

**基准卡片尺寸：** 520×940

**缩放因子：**
```go
scale = 当前卡片宽度 / 520.0
```

**字体大小计算：**
```go
实际字体大小 = 基准字体大小 × scale
```

**间距计算：**
```go
实际间距 = 基准间距 × scale
```

### 坐标计算

**卡片位置：**
```go
x = padding + col*(cardWidth+padding)
y = headerHeight + padding + row*(cardHeight+padding)
```

**元素位置（卡片内）：**
```go
元素Y坐标 = 卡片Y + 基准偏移 × scale
元素X坐标 = 卡片X + 基准偏移 × scale
```

## 💡 使用建议

### 最佳实践

1. **小批量报表（1-6条）**
   - 适合：单户详细查看
   - 打印：A4纸张
   - 分享：微信/邮件

2. **中批量报表（7-12条）**
   - 适合：楼层汇总
   - 打印：A3纸张
   - 分享：在线查看

3. **大批量报表（13+条）**
   - 适合：整栋楼汇总
   - 打印：建议分页
   - 分享：PDF格式

### 性能优化建议

1. **超过30条记录**
   - 考虑分批生成
   - 或使用筛选功能

2. **网络分享**
   - 压缩PNG图片
   - 或转换为PDF

3. **长期存档**
   - 定期清理旧报表
   - 保留JSON备份
