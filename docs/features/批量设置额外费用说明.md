# 批量设置额外费用功能说明

## 📝 功能概述

批量设置额外费用功能允许您一次性为多个账单记录添加相同的额外费用项。这在需要统一收取某些费用（如公共区域维护费、季度检修费等）时非常有用。

---

## 🎯 功能特点

- ✅ **批量操作** - 可同时为多条记录设置额外费用
- ✅ **灵活模式** - 支持追加和替换两种模式
- ✅ **多项费用** - 可以同时添加多个额外费用项
- ✅ **即时生效** - 操作完成后立即更新总费用
- ✅ **选择灵活** - 可以任意选择需要设置的记录

---

## 📖 使用方法

### 1️⃣ 选择记录

在表格中勾选需要设置额外费用的记录复选框。

```
☑ 101 - 2025-11
☑ 102 - 2025-11
☑ 201 - 2025-11
☐ 202 - 2025-11
```

**提示：** 可以使用表头的全选复选框快速选择所有记录。

### 2️⃣ 打开批量设置界面

点击控制面板中的 **"💵 批量设置额外费用"** 按钮。

### 3️⃣ 添加额外费用项

1. 点击 **"➕ 添加费用项"** 按钮
2. 输入费用名称（如："公共区域维护费"）
3. 输入金额（如：50.00）
4. 可继续添加多个费用项

**示例：**
```
费用名称：公共区域维护费    金额：50.00
费用名称：垃圾清运费        金额：20.00
费用名称：电梯维护费        金额：30.00
```

### 4️⃣ 选择操作模式

#### 追加模式（默认）
- 在现有额外费用基础上追加新的费用项
- 原有的额外费用不会被删除
- 适合：定期添加新的费用项

**示例：**
```
原有费用：水管维修费 ¥100
追加费用：公共清洁费 ¥30
结果：    水管维修费 ¥100 + 公共清洁费 ¥30
```

#### 替换模式
- 清空所有现有额外费用，替换为新的费用项
- 原有的额外费用会被完全删除
- 适合：重新规划费用结构

**示例：**
```
原有费用：水管维修费 ¥100 + 电路检修费 ¥80
替换费用：公共清洁费 ¥30
结果：    公共清洁费 ¥30
```

### 5️⃣ 执行操作

点击 **"应用"** 按钮，系统将：
1. 为所有选中的记录设置额外费用
2. 自动重新计算总费用
3. 更新表格显示
4. 显示操作结果提示

## 🔧 API 接口

### 请求

**端点：** `POST /api/v1/billing/batch-extra-fee`

**请求体：**
```json
{
  "ids": [1, 2, 3],
  "extraFees": [
    {
      "name": "公共区域维护费",
      "amount": 50.00
    },
    {
      "name": "垃圾清运费",
      "amount": 20.00
    }
  ],
  "mode": "append"
}
```

**参数说明：**
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| ids | array | 是 | 要设置额外费用的记录ID列表 |
| extraFees | array | 是 | 额外费用项列表 |
| extraFees[].name | string | 是 | 费用名称 |
| extraFees[].amount | number | 是 | 费用金额 |
| mode | string | 否 | 操作模式：`append`（追加）或 `replace`（替换），默认为 `append` |

### 响应

**成功：**
```json
{
  "success": true,
  "message": "批量设置成功",
  "count": 3
}
```

**失败：**
```json
{
  "success": false,
  "error": "请选择要设置额外费用的记录"
}
```

## 💡 使用场景

### 场景1：每月公共费用

每月月初为所有住户统一添加公共区域维护费。

```
操作步骤：
1. 全选当月所有记录
2. 添加"公共区域维护费"
3. 选择"追加模式"
4. 执行操作
```

### 场景2：季度检修费

每季度为特定楼层的住户添加电梯维护费。

```
操作步骤：
1. 筛选特定月份
2. 选择特定楼层的住户（如201-205）
3. 添加"电梯维护费"
4. 选择"追加模式"
5. 执行操作
```

### 场景3：费用调整

发现之前添加的额外费用有误，需要重新设置。

```
操作步骤：
1. 选择需要调整的记录
2. 添加正确的额外费用项
3. 选择"替换模式"
4. 执行操作
```

### 场景4：批量清除额外费用

需要清除某些记录的所有额外费用。

```
操作步骤：
1. 选择需要清除的记录
2. 不添加任何费用项（或添加金额为0的项）
3. 选择"替换模式"
4. 执行操作
```

## ⚠️ 注意事项

### 操作不可撤销
- 批量设置操作完成后不可撤销
- 建议在重要操作前先导出备份数据
- 可使用"预览"功能确认选择的记录

### 费用计算
- 系统会自动重新计算总费用
- 包含：管理费 + 水费 + 电费 + 所有额外费用
- 更新后可在表格中查看新的总费用

### 模式选择
- **追加模式**：不会删除原有额外费用
- **替换模式**：会清空所有原有额外费用
- 请根据实际需求谨慎选择

### 数据验证
- 费用名称不能为空
- 费用金额必须大于0
- 至少需要选择一条记录
- 至少需要添加一项额外费用

## 🎬 完整示例

### PowerShell 脚本示例

```powershell
# 批量设置额外费用示例

$baseUrl = "http://localhost:8080/api/v1"

# 准备数据
$data = @{
    ids = @(1, 2, 3, 4, 5)  # 记录ID列表
    extraFees = @(
        @{
            name = "公共区域维护费"
            amount = 50.00
        },
        @{
            name = "垃圾清运费"
            amount = 20.00
        },
        @{
            name = "电梯维护费"
            amount = 30.00
        }
    )
    mode = "append"  # 追加模式
} | ConvertTo-Json -Depth 10

# 执行批量设置
try {
    $result = Invoke-RestMethod -Uri "$baseUrl/billing/batch-extra-fee" `
        -Method POST `
        -ContentType "application/json" `
        -Body $data
    
    if ($result.success) {
        Write-Host "✓ 成功为 $($result.count) 条记录设置额外费用！" -ForegroundColor Green
    } else {
        Write-Host "✗ 操作失败: $($result.error)" -ForegroundColor Red
    }
} catch {
    Write-Host "✗ 请求失败: $($_.Exception.Message)" -ForegroundColor Red
}
```

### JavaScript 示例（前端）

```javascript
// 批量设置额外费用
async function batchSetExtraFees() {
    const selectedIds = [1, 2, 3]; // 选中的记录ID
    
    const data = {
        ids: selectedIds,
        extraFees: [
            { name: "公共区域维护费", amount: 50.00 },
            { name: "垃圾清运费", amount: 20.00 }
        ],
        mode: "append"
    };
    
    try {
        const response = await fetch('http://localhost:8080/api/v1/billing/batch-extra-fee', {
            method: 'POST',
            headers: {
                'Content-Type': 'application/json'
            },
            body: JSON.stringify(data)
        });
        
        const result = await response.json();
        
        if (result.success) {
            console.log(`成功为 ${result.count} 条记录设置额外费用！`);
        } else {
            console.error('操作失败:', result.error);
        }
    } catch (error) {
        console.error('请求失败:', error);
    }
}
```
