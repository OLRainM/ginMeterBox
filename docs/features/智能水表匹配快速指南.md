# 🎯 智能水表匹配 - 快速使用指南

## 5分钟上手

### 场景描述

假设你是物业管理员，刚从4个房间抄完水表，记录了以下读数：
```
2893, 2870, 2920, 2900
```

但是你忘记标注每个读数对应哪个房号了！😱

别担心，智能水表匹配功能可以帮你自动匹配！

---

## 使用步骤

### 1️⃣ 选择用户

在主界面勾选需要匹配的4个用户：

```
☑️ 101号房 (上月水表: 2870)
☑️ 102号房 (上月水表: 2850)
☑️ 103号房 (上月水表: 2900)
☑️ 104号房 (上月水表: 2880)
```

### 2️⃣ 打开智能匹配

点击 **"🎯 智能水表匹配"** 按钮

### 3️⃣ 输入读数

在文本框中输入你抄的水表读数：
```
2893, 2870, 2920, 2900
```

或者用空格分隔：
```
2893 2870 2920 2900
```

或者换行输入：
```
2893
2870
2920
2900
```

### 4️⃣ 预览结果

点击 **"🔍 预览匹配结果"**

系统会显示最优匹配方案：

```
🎯 最优匹配方案（总用水量最小）
总用水量：83 吨

101号房: 2870 → 2893 (用量: 23吨)
102号房: 2850 → 2870 (用量: 20吨)
103号房: 2900 → 2920 (用量: 20吨)
104号房: 2880 → 2900 (用量: 20吨)
```

### 5️⃣ 确认应用

确认无误后，点击 **"✅ 确认并应用"**

完成！系统会自动更新所有用户的水表读数和水费。

---

## 为什么这样匹配？

系统尝试了所有可能的匹配组合（共24种），计算每种组合的总用水量：

| 匹配方案 | 101 | 102 | 103 | 104 | 总用水量 |
|---------|-----|-----|-----|-----|---------|
| 方案1   | 2893| 2870| 2920| 2900| **83吨** ✅ |
| 方案2   | 2893| 2870| 2900| 2920| 103吨 |
| 方案3   | 2870| 2893| 2920| 2900| 103吨 |
| ...     | ... | ... | ... | ... | ... |

系统选择了总用水量最小的方案（83吨），因为这最符合实际情况。

## 实用技巧

### 💡 技巧1：复制粘贴

从Excel或记事本直接复制粘贴读数：
```

2920
2900
```

### 💡 技巧2：混合分隔

可以混用多种分隔符：
```
2893, 2870
2920 2900
```

### 💡 技巧3：分批处理

如果用户很多（>10个），建议分批处理：
- 第一批：101-105号房
- 第二批：106-110号房
- ...

### 💡 技巧4：验证结果

匹配后检查用水量是否合理：
- 正常家庭：15-30吨/月
- 异常大：>50吨/月（可能匹配错误）
- 异常小：<5吨/月（可能匹配错误）

## 常见问题

### ❓ 读数数量不匹配怎么办？

**错误提示：** "读数数量(3)与选中用户数量(4)不匹配"

**解决方法：**
1. 检查是否漏输入了某个读数
2. 检查是否多选了用户
3. 确保读数和用户数量一致

### ❓ 匹配结果不理想？

**可能原因：**
1. 上月读数不准确
2. 补差值设置错误
3. 输入的读数有误

**解决方法：**
1. 检查上月读数是否正确
2. 使用编辑功能手动调整
3. 重新输入读数再次匹配

### ❓ 可以撤销匹配吗？

匹配操作会直接更新数据库，无法自动撤销。

**建议：**
1. 匹配前仔细预览结果
2. 如需撤销，手动编辑每条记录
3. 或者使用导入功能恢复备份数据

## 适用场景

### ✅ 适合使用

- 抄表时忘记标注房号
- 批量录入多个用户数据
- 需要快速完成数据录入
- 读数顺序与房号顺序不一致

### ❌ 不适合使用

- 只有1-2个用户（直接编辑更快）
- 读数已经明确对应房号
- 用户数量过多（>15个，建议分批）
