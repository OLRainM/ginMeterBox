# 筛选和批量延续功能说明

## 📅 更新日期
2025年11月29日

---

## 🎯 新增功能

### 1. 按房号筛选功能
在月份筛选的基础上，新增房号筛选，支持组合筛选。

### 2. 批量自动延续功能
自动延续功能扩展，支持单户和批量两种模式，可以一次性为多个住户创建新月份记录。

---

## 📋 功能详情

### 一、按房号筛选

#### 功能位置
控制面板 > 筛选区域 > 按房号筛选下拉框

#### 使用方法

1. **单独筛选房号**
   - 选择房号下拉框中的任意房号
   - 表格将只显示该房号的所有记录

2. **组合筛选（月份 + 房号）**
   - 先选择月份：`2025-11`
   - 再选择房号：`101`
   - 表格将只显示该房号在该月份的记录

3. **清除筛选**
   - 点击"清除筛选"按钮
   - 恢复显示所有记录

#### 功能特点

- ✅ **智能填充** - 下拉框自动填充所有存在的房号
- ✅ **自动排序** - 房号按升序排列
- ✅ **实时更新** - 数据变化时自动更新下拉框
- ✅ **保持状态** - 刷新数据后保持选中状态
- ✅ **组合筛选** - 可与月份筛选同时使用

#### 使用场景

| 场景 | 操作 | 效果 |
|------|------|------|
| 查看某住户所有账单 | 选择房号 | 显示该房号的历史记录 |
| 查看某月某户账单 | 选择月份+房号 | 精准定位单条记录 |
| 批量查看某月账单 | 只选择月份 | 显示该月所有住户 |

### 二、批量自动延续

#### 功能位置
点击"🔄 自动延续"按钮 > 选择批量模式

#### 两种模式

##### 1️⃣ 单户模式（默认）
- **适用场景**：为单个住户创建新记录
- **操作步骤**：
  1. 选择"单户模式"
  2. 输入住户编号
  3. 选择新月份
  4. 点击"预览"查看上月数据（可选）
  5. 点击"创建"完成

##### 2️⃣ 批量模式（新增）⭐
- **适用场景**：为多个住户批量创建新记录
- **操作步骤**：
  1. 选择"批量模式"
  2. 在住户列表中勾选需要的房号
     - 可使用"全选"快速选择所有
     - 可使用"取消全选"清除选择
     - 实时显示已选择数量
  3. 选择新月份
  4. 点击"创建"批量完成

#### 功能特点

**单户模式：**
- ✅ 精准控制
- ✅ 支持预览上月数据
- ✅ 详细的数据展示

**批量模式：**
- ✅ 高效批量操作
- ✅ 可视化选择界面
- ✅ 全选/取消全选快捷操作
- ✅ 实时显示选择数量
- ✅ 部分成功处理
- ✅ 失败项目反馈

#### 自动延续规则

系统将从上月记录自动继承以下数据：

| 数据项 | 上月 | 新记录 |
|--------|------|--------|
| 水表读数 | currentWater | previousWater 和 currentWater |
| 电表读数 | currentElectric | previousElectric 和 currentElectric |
| 管理费 | managementFee | managementFee |
| 水单价 | waterPrice | waterPrice |
| 电单价 | electricPrice | electricPrice |
| 额外费用 | extraFees | 不继承（需手动添加） |

**说明：**
- 新记录的本月读数初始化为上月的当前读数
- 用户可以后续修改新记录的读数
- 额外费用不会自动继承，避免重复收费

## 🔌 API 接口

### 批量自动延续接口

**端点：** `POST /api/v1/billing/batch-continue`

**请求体：**
```json
{
  "roomNumbers": ["101", "102", "201"],
  "newMonth": "2025-12"
}
```

**参数说明：**
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| roomNumbers | array | 是 | 住户编号列表 |
| newMonth | string | 是 | 新月份（YYYY-MM格式） |

**成功响应：**
```json
{
  "success": true,
  "count": 3,
  "message": "成功为 3 个住户创建新记录"
}
```

**部分成功响应：**
```json
{
  "success": true,
  "count": 2,
  "partialSuccess": true,
  "failed": ["201"],
  "message": "成功为 2 个住户创建新记录，1 个失败"
}
```

**失败响应：**
```json
{
  "success": false,
  "error": "批量自动延续失败，所有住户都未能成功创建",
  "failed": ["101", "102", "201"]
}
```

## ⚠️ 注意事项

### 房号筛选

1. **数据同步**
   - 房号下拉框在加载数据时自动更新
   - 刷新数据后保持当前选择（如果该房号仍存在）

2. **筛选优先级**
   - 先应用月份筛选
   - 再应用房号筛选
   - 两个条件同时生效

3. **清除筛选**
   - 一次性清除所有筛选条件
   - 恢复显示完整数据

### 批量自动延续

1. **数据来源**
   - 必须存在上月的记录才能延续
   - 系统自动查找每个房号的最新记录

2. **失败处理**
   - 如果某些住户没有历史记录，会跳过并标记为失败
   - 成功的住户仍会创建新记录
   - 最后显示成功和失败的统计

3. **数据验证**
   - 至少选择一个住户（批量模式）
   - 新月份不能为空
   - 房号不能重复

4. **额外费用**
   - 额外费用**不会**自动延续
   - 避免重复收取一次性费用
   - 如需添加，请在创建后手动添加
