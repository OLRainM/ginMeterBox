# 🎯 智能水表匹配功能说明

## 功能概述

智能水表匹配是一个创新的批量数据录入功能，允许用户选择多个住户后，批量输入一组水表读数，系统会自动按照**最小总用水量**的原则将这些读数智能匹配给各个住户。

## 使用场景

在实际抄表过程中，可能会遇到以下情况：
- 抄表员记录了一组水表读数，但没有明确标注每个读数对应的房号
- 需要快速录入多个住户的水表数据
- 希望系统自动优化匹配，减少人工判断

## 核心算法

系统采用**全排列最优化算法**：
1. 生成所有可能的水表读数与住户的匹配组合
2. 计算每种组合的总用水量
3. 选择总用水量最小的组合作为最优匹配方案

### 计算公式

对于每个住户：
```
用水量 = 本月水表读数 - 上月水表读数 + 水表补差
```

对于整体方案：
```
总用水量 = Σ(所有住户的用水量)
```

系统会选择使总用水量最小的匹配方案。

## 使用步骤

### 1. 选择住户

在主界面的数据表格中，勾选需要匹配的住户记录。

**注意：** 选择的住户数量必须与后续输入的水表读数数量一致。

### 2. 打开智能匹配

点击操作面板中的 **"🎯 智能水表匹配"** 按钮。

### 3. 查看选中信息

模态框会显示已选中的住户信息，包括：
- 房号
- 上月水表读数
- 水表补差值

### 4. 输入水表读数

在文本框中输入水表读数，支持多种分隔方式：
- **空格分隔**：`2893 2870 2850 2920`
- **逗号分隔**：`2893, 2870, 2850, 2920`
- **换行分隔**：
  ```
  2893
  2870
  2850
  2920
  ```
- **混合分隔**：`2893, 2870 2850, 2920`

### 5. 预览匹配结果

点击 **"🔍 预览匹配结果"** 按钮，系统会：
- 验证读数数量是否匹配
- 计算最优匹配方案
- 显示详细的匹配结果

预览界面会显示：
- 总用水量
- 每个住户的匹配详情（房号、上月读数、本月读数、用水量）

### 6. 确认并应用

确认匹配结果无误后，点击 **"✅ 确认并应用"** 按钮。

系统会：
- 更新所有选中住户的本月水表读数
- 自动重新计算水费
- 刷新数据表格

## 注意事项

### ⚠️ 数量匹配
- 水表读数数量必须与选中住户数量完全一致
- 系统会在预览前进行验证

### ⚠️ 数据有效性
- 确保输入的都是有效的数字
- 非数字内容会被自动过滤

### ⚠️ 算法复杂度
- 对于少量住户（≤10个），算法运行非常快
- 住户数量较多时（>10个），计算时间会增加
- 建议分批处理大量数据

### ⚠️ 数据备份
- 匹配操作会直接更新数据库
- 建议在操作前确认数据正确性
- 可以通过编辑功能手动调整

## 技术实现

### 后端（Go）

```go
// 智能匹配核心算法
func smartMatchWaterReadings(records []*models.BillingRecord, readings []float64) []WaterMatch {
    // 生成所有排列组合
    permutations := generatePermutations(readings)
    
    // 寻找最小总用水量的方案
    minTotalUsage := float64(1e18)
    var bestMatches []WaterMatch
    
    for _, perm := range permutations {
        totalUsage := 0.0
        for i := 0; i < len(records); i++ {
            usage := perm[i] - records[i].PreviousWater + records[i].WaterAdjustment
            totalUsage += usage
        }
        
        if totalUsage < minTotalUsage {
            minTotalUsage = totalUsage
            // 更新最佳方案
        }
    }
    
    return bestMatches
}
```

### 前端（JavaScript）

```javascript
// 本地模拟匹配（用于预览）
function simulateMatch(records, readings) {
    const permutations = generatePermutations(readings);
    let minTotalUsage = Infinity;
    let bestMatches = [];
    
    for (const perm of permutations) {
        let totalUsage = 0;
        const currentMatches = [];
        
        for (let i = 0; i < records.length; i++) {
            const usage = perm[i] - records[i].previousWater + 
                         (records[i].waterAdjustment || 0);
            totalUsage += usage;
            currentMatches.push({ record: records[i], waterReading: perm[i], waterUsage: usage });
        }
        
        if (totalUsage < minTotalUsage) {
            minTotalUsage = totalUsage;
            bestMatches = currentMatches;
        }
    }
    
    return { matches: bestMatches, totalUsage: minTotalUsage };
}
```

## 常见问题

### Q1: 为什么要按最小总用水量匹配？

**A:** 在正常情况下，水表读数应该是递增的，用水量应该在合理范围内。最小总用水量原则可以：
- 避免异常大的用水量
- 减少匹配错误的可能性
- 符合实际用水规律

### Q2: 如果匹配结果不理想怎么办？

**A:** 可以通过以下方式调整：
1. 使用编辑功能手动修改个别记录
2. 重新选择住户和读数进行匹配
3. 检查上月读数和补差值是否正确

### Q3: 支持电表匹配吗？

**A:** 当前版本仅支持水表匹配。如需电表匹配功能，可以参考本功能进行扩展开发。

### Q4: 匹配算法的时间复杂度是多少？

**A:** 时间复杂度为 O(n! × n)，其中 n 是住户数量。建议：
- n ≤ 8：瞬间完成
- n = 9-10：1秒内完成
- n > 10：建议分批处理
